# ClickHouse Events Analytics
ClickHouse Events Analytics
OLAP аналитика пользовательских событий с расчетом Retention в ClickHouse.

📊 Результаты выполнения
После выполнения скрипта вы получите:

Агрегированные данные по дням и типам событий

Расчет 7-дневного Retention

Общую статистику по пользователям и событиям

📈 Ключевые метрики:
Уникальные пользователи: 6

Сумма потраченных баллов: 295

Всего событий: 20

Retention: 66.67% - 100%

🏗 Архитектура проекта
Базы данных:
user_events - таблица сырых событий (TTL 30 дней)

user_events_aggregated - агрегированные данные (TTL 180 дней)

user_events_mv - материализованное представление для автоматической агрегации

Ключевые технологии:
Materialized Views - для автоматической агрегации данных

State/Merge функции - uniqState, sumState, countState

AggregatingMergeTree - для хранения агрегированных данных

TTL (Time To Live) - автоматическая очистка старых данных

📋 Основные компоненты
1. Таблицы
Сырые события - хранение исходных данных пользователей

Агрегированные данные - предрасчитанные метрики по дням и типам событий

2. Materialized View
Автоматически обновляется при вставке новых данных

Выполняет агрегацию в реальном времени

Использует State-функции для эффективных вычислений

3. Аналитические запросы
Быстрая аналитика через Merge-функции

Расчет Retention пользователей

Общая статистика активности

🔧 Использование

select 
	event_date,
	event_type, uniqMerge(total_users) as uniq_users, 
	sumMerge(total_spent_points) as total_spent,
	countMerge(total_actions) as actions_count
from user_events_aggregated
group by event_date, event_type 
order by event_date desc, event_type;

-- Запрос автоматически рассчитывает 7-дневный Retention
-- Формат результата: total_users_day_0|returned_in_7_days|retention_7d_percent|

📊 Примеры результатов
Retention:
2025-10-24: 3|2|66.67%  (3 пользователя → 2 вернулись)
2025-10-27: 3|3|100%    (3 пользователя → 3 вернулись)

Агрегированные данные:
По дням и типам событий с количеством пользователей, суммой баллов и количеством действий.

🎯 Особенности реализации
Эффективность: State-функции позволяют инкрементально обновлять агрегаты

Производительность: AggregatingMergeTree оптимизирован для аналитических запросов

Автоматизация: MV исключает необходимость ручного пересчета агрегатов

Масштабируемость: TSL обеспечивает автоматическое управление данными
